What You Need To Know

API Access Claims are an important administrative service in OrderCloud. This service is where you will explicitly create each application instance and where you will define the scope of each application's access rights. Without this you would not be able to authenticate as a buyer user, which would prove troubling within your buyer applications.

When processing a successful API Access Claim, an ID will be returned to you (of which you can optionally define in the request yourself) that will act as your applications clientID. This is what you will store in the base of your application as an app constant named clientid. This clientID is what the client application will use to tell OrderCloud what buyer environment it should be looking under when a user attempts to authenticate on the site.

Creating a New User

A full API Access Request Object is shown below. We will only be using part of this object, as the ClientSecret is irrelevant for this call (we'll be using the administrative token stored in your browsers cookies for your current admin user), and DefaultUserContextID is an advanced topic we won't be touching on right now.

You can read more about the code in the documentation, but for now just set your the AppName of your choosing and leave everything else the same. Note that the AccessTokenDuration and RefreshTokenDuration are in milliseconds, and you are currently making an application that has 'FullAccess' claims, which means that a buyer-user has the same level of access to the system as an admin user (they can create products, categories, users, etc., just like your admin user). Don't worry about access levels right now, as we are just focused on building a basic environment right now.

When you are ready, go ahead and fill in the AppName value in the claim object and press submit.
