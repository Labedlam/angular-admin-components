What You Need To Know

Before we create the final necessary assignments, it is necessary that we create a category. Beyond providing a way to keep products organized and grouped, a category also acts as an access-point for buyer-users to access a product or set of products. A user can only access a product through a category that has the product assigned to it. Additionally, a user has to be assigned to that category in order to gain access to the product. In other words, there are two category assignments necessary to create an indirect relationship between a user and product.

Before we get into those assignments, let's create a category that you will use for this exercise.

Creating a category is much like creating a user--it must be defined under the context of a buyer, and requires a request object. Go ahead and set the buyerID variable with the ID of the buyer you created in the first class. Now take a look at the cat variable.

Let's start by taking a few of the values we don't need out. Remove the ParentID and the second XP (the last value) values, as the the former is outside the scope of this course, and the latter isn't supposed to be in there (to be fixed). Then, go ahead and fill out the ID, Name, and Description out with anything you would like. You will also want to leave xp null, and ListOrder set to 1. Finally, set Active to true. This needs to be set to true or the category won't be accessible on a buyer application!

You only need to create one category, so go ahead and run when you are ready!
