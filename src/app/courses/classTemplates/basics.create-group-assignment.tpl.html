What You Need To Know

Now for the final step of creating and using a group, assigning users to the user group. The typical first steps towards accessing any object as a buyer-user is to create the necessary objects and then assign them. This is the first of many create and assign combinations you will be executing. In fact, you will be creating three more objects and completing three more assignments in this course alone.

The reason assignments are explicitly defined outside of object creations is because OrderCloud is built as a modular API. Having to create and assign separately may seem like more work, but there is a very good reason for it. We want you to use exactly what you need to build an application for your company or your clients, and nothing more. If we started creating nested objects directly inside of other objects (such as products inside of a category or prices inside of a product), it would require you to use more than you really need, thus slowing down the development time and vastly reducing your ability to define a proper project scope (this is the crux of a SAAS application). And as an added plus, you will come to love the modular orientation of OrderCloud once you get deeper into your application because the separation of concerns makes it easier to manage and intuitively understand the structure of your B2B environment.

Creating a Group Assignment

A full group assignment object is shown below,

Taking a look at the create.js script, you can see what is required to create a new Group assignment. Start by replacing the buyerID variable with the ID of the new buyer you created earlier in this course. In the assignment variable, a sample request object is currently stored with 3 values--UserGroupID, UserID, and BuyerID. You can take out the BuyerID object, as we won't need that (and it is already stated in the parameter value).

You will be assigning two of your three new users to this group. You have two options to accomplish this. Your first option is to process the create.js script twice with each of the users you are assigning. The second and more challenging option is to create a second assignment object (with a different variable name, of course) and process a second SDK call in the same script with the new assignment object.

If you are ready, go ahead and process the request by hitting "Run".
