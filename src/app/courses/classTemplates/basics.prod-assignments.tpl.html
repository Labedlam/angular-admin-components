What You Need To Know

Now that you have a product, price schedule, and user/group defined and ready to work with, let's complete the final step to make your first 'product instance'. The classes you have completed so far have created the necessary pieces to make a proper product assignment that creates a relationship between the three, and ultimately create a 'product instance'. After this class, we will complete building your first catalog by creating a category and making the necessary assignments so your new users can actually access the products they are supposed to have access to.

Creating a Product Assignment

In the create.js script, you will see a variable, assignment, and the SDK call. As you can see in this variable, there are six different values. We will only be using three of the six to create the assignment. To learn more about all of these values, review the documentation.

Let's start by removing the values we don't need. Remove ReplenishmentPriceScheduleID, UserID, and BuyerID. Now that you have the three values you need--ProductID, StandardPriceSchedule, and UserGroupID--you are ready to set the assignments. Fill out the StandardPriceSchedule with your newly created Price Schedule ID. Do the same with the UserGroupID with your newly created UserGroup ID.

Before you fill out the ProductID, you should know that you will need to create 3 different product assignments in this class, for each of the 3 products that you created in an earlier class. Go ahead and fill out the ProductID object with each of these 3 new product's ID's and process the create.js for each one.
