What You Need To Know

The next step towards creating a catalog is to create a few products. What makes a product unique in terms of its orientation is that it is not defined under the context of a buyer, like a user or a category (as you will see shortly). This buyer autonomy means that you can use one product across multiple buyers without having to create a new one.

You will get a hang of these idiosyncrasies as you continue to build your B2B environment with OrderCloud. For now, we will focus on creating your first product, followed by creating the necessary assignments to make the product accessible.

Creating a Product

Below is a sample of a new product request object,

We will focus on the create.js script. As you can see, there is an entire product object set inside the prod variable. To create the new product, you only need to set the ID, Description, Name, QuantityMultiplier, and Active values. Everything else is up to you, but we recommend you focus on the fundamentals right now.

Set the ID, Description, and Name to whatever you like. Set the QuantityMultiplier to 1 or more, and set Active to true. Note that in order for a buyer-user to access a product, it MUST be active.

Now that you have completed the configuration, review the code block. Note that the service call to create a Product is, Products.Create(prod);, where "prod" is the new object you are creating, and the only parameter in the method call.

You will need to create 3 new products in order to successfully complete this class. Go ahead and create 3 new products when you are ready. Make sure that each of the new products has its own separate ID. You should also write down the ID of each of your new products to be used later. If you'd like, you can use the list.js script to generate the list of products you have created once you are done creating your 3 new products.
