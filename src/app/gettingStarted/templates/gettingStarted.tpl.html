Getting Started

This guide is geared toward developers who want to get up and running with the OrderCloud API and start creating their first application with the OrderCloud Seed repository. While your development environment is up to you, Four51's IDE of choice is WebStorm, which works well with all of the tools below.

Requirements

The OrderCloud Seed depends on a few third party code libraries that you will need to install locally before you can jump into the code. It isn't necessary to learn the full workings of any of these libraries to get started, only to install them.

Four51's IDE of choice is WebStorm, which works well with all of the tools below.

Installing Node.js and NPM

  1. Follow the link above to the Node.js home page. This page should display green download buttons specific for the OS you are currently using. At the time of writing this guide Node.js has two versions available, a long term support (LTS) version 4.2.2 and a staple version of 5.0.0. Either version should be fine for working with our application. Click on the appropriate button to begin downloading Node.js.
  2. After the application has finished downloading, click on the downloaded file and walk though the installation guide. All the defaults should be fine.
  3. The Node.js installer should take care of installing NPM as well. If you want to verify the install, open up a terminal or command prompt window and type node --version and your current version of Node.js should be returned. You can do the same check with NPM by typing npm --version. If there was no error during your installation but these commands do not work, you may need to add node and/or npm to your environment variables.

Installing Bower

The seed uses Bower to add 3rd-party code libraries to your application. Examples include AngularJS, Bootstrap, and the OrderCloud AngularJS SDK.

To install Bower, just run npm install -g bower.

Make sure you don't forget the "-g", this is telling NPM to install the package globally so it will be available to you for all future projects.

Installing Gulp 4.0

The seed relies on Gulp for its automated build/test process. It uses the 4.0 version of Gulp which at the time of writing this guide has not been officially released, but fear not! NPM makes installing this version a breeze:

npm -g install "gulpjs/gulp-cli#4.0"

Again make sure you include the "-g". Now that you have all the third party stuff installed, you are finally ready to get the OrderCloud Seed code.

Getting the Seed

The OrderCloud Seed is freely available on GitHub. If you don't have a GitHub account yet you are going to need to create one. (Don't worry, it's free.)

The OrderCloud Seed repository is located here: https://github.com/Four51/OrderCloud-Seed-AngularJS.

Learning Git and GitHub is beyond the scope of this guide, but GitHub provides a wealth of resources to help you get up to speed. The remainder of this guide will assume that you have successfully pulled a copy of the OrderCloud Seed repository to a local directory.

Final Steps

It's the home stretch. Just a few more things to do and your app will be up and running.

To ensure your setup works, build your application and then run it with the following commands:

gulp build

gulp watch

The built files are placed in the build/ directory by default. watch should open the application on localhost, port 8000.

watch actually starts a few other processes in the background to help you develop your application. Using browser-sync and some built in Gulp functions, the app is now watching for changes in your source directory. If you make any changes to your html or js files, the app should automatically reload your application with the appropriate changes. And if you make any changes to your style sheets (CSS or LESS) the app will rebuild those changes and inject them directly into the application, without reloading the entire page!

When you're ready to push your app into production, just run the compile command:

gulp compile

This will concatenate and minify your sources and place them by default into the compile/ directory. There will only be three files (excluding assets): index.html, OrderCloud.js, and OrderCloud.css. All of the vendor dependencies like AngularJS styles and the OrderCloud SDK will have been merged into those files for super-easy deploying. If your application includes any assets (images, etc. under src/assets/), those will be copied to compile/ as is.

Lastly, a complete build is always available by simply running the default task, which runs build and then compile:

gulp
